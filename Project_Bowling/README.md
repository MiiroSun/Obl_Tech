### Боулинг-центр: Задача на конкурентность и каналы в Go

#### Цель задачи
**Разработать систему управления боулинг-центром с диспетчером, который распределяет очередь клиентов по ограниченному пулу дорожек, демонстрируя работу с каналами и конкурентностью в Go.**

#### Функциональные требования:
**Клиенты**
- [x] Приходят последовательно с течением времени
- [x] Каждый клиент имеет: ID, ожидаемое время игры
- [x] Могут уйти если ждут слишком долго

**Дорожки**
- [x] Фиксированный пул (например, 5 дорожек)
- [x] Работают параллельно и независимо
- [x] Могут быть либо заняты (один клиент), либо свободны
- [x] После завершения игры освобождаются
- [x] Разные игры имеют разную длительность

**Диспетчер (Manager)**
- [x] Принимает новых клиентов
- [x] Проверяет наличие свободных дорожек
- [x] Назначает клиента на свободную дорожку или ставит в очередь ожидания
- [x] Когда дорожка освобождается, берет следующего клиента из очереди
- [x] Ведет учет: сколько клиентов ждут, сколько играют сейчас, сколько завершили игры

**Табло (то, что отображается в консоли)**
- [ ] Отображает текущий счет и активного клиента по всем активным дорожкам
- [x] Когда завершается день (все клиенты закончились), выводит статистику по всем игрокам

**Для гуру** — реализовать веб-интерфейс

**Технические ограничения**

- [x] Размер пула дорожек ― константа (задается при старте системы)
- [x] Каждая сущность (клиент, дорожка, диспетчер, табо) "живут в своей горутине"
- [x] Конкурентность: дорожки работают параллельно, клиенты обрабатываются независимо
- [x] Очередь ожидания: клиенты ждут в порядке поступления
- [x] Использование каналов для коммуникации между компонентами
- [ ] Безопасность при работе с общими данными
- [ ] Корректное завершение работы всех горутин

**Пример бизнес-процесса**
Рассмотрим сценарий с 8 клиентами и пулом из 5 дорожек.

*Фаза 1:* Начало работы

- Система запускается. Все 5 дорожек свободны.

*Фаза 2:* Заполнение пула

- Клиенты 1-5 приходят один за другим и немедленно получают дорожки. Все дорожки заняты, очередь пуста.

*Фаза 3:* Формирование очереди

- Клиенты 6, 7, 8 прибывают. Все дорожки заняты, поэтому они попадают в очередь ожидания в порядке прибытия.

*Фаза 4:* Освобождение и перераспределение

- Клиент 1 завершает игру на дорожке 1. Дорожка освобождается. Диспетчер автоматически берет Клиента 6 из очереди и назначает на освободившуюся дорожку 1.

- Когда дорожка 3 освобождается, Клиент 7 получает ее. Когда дорожка 2 освобождается, Клиент 8 получает ее.

*Фаза 5:* Завершение

- Все клиенты завершили игры, все дорожки свободны, очередь пуста.

**Критерии успеха**

- [x] Система обрабатывает клиентов в правильном порядке (FIFO для очереди)
- [x] Никогда не бывает более N активных игр одновременно (N = размер пула)
- [x] Очередь корректно уменьшается при освобождении дорожек
- [x] Статистика корректна несмотря на параллельные операции
- [ ] Табло отображает все корректно и красиво
